# Full Chat Screen Implementation

## Overview

Chat screen yang mengintegrasikan semua library chat dengan DuckDB persistence.

## Libraries Terintegrasi

| Library | Status | Keterangan |
|---------|--------|------------|
| `@lib/data/` | ✅ | DuckDB persistence layer |
| `@lib/flutter_chat_core/` | ✅ | Core models & ChatController |
| `@lib/flutter_chat_ui/` | ✅ | UI components |
| `@lib/flyer_chat_text_message/` | ✅ | Text messages dengan markdown |
| `@lib/flyer_chat_system_message/` | ✅ | System messages |
| `@lib/flyer_chat_text_stream_message/` | ✅ | Streaming text messages |
| `@lib/flyer_chat_file_message/` | ✅ | File messages |

## File Structure

```
lib/
├── main.dart                        # App entry point
├── screens/
│   └── full_chat_screen.dart        # Main chat screen
├── data/                            # Data layer (DuckDB)
├── flutter_chat_core/               # Core models
├── flutter_chat_ui/                 # UI components
├── flyer_chat_text_message/         # Text message widget
├── flyer_chat_system_message/       # System message widget
├── flyer_chat_text_stream_message/  # Stream message widget
└── flyer_chat_file_message/         # File message widget
```

## Features

### 1. Text Messages
- Markdown support via `gpt_markdown`
- Link preview support
- Reactions support
- Pin/unpin messages
- Delete messages

### 2. System Messages
- Informational messages (e.g., "User joined the chat")
- Full-width centered display

### 3. Text Stream Messages
- AI response streaming support
- Animated opacity per chunk
- Markdown rendering

### 4. File Messages
- File name and size display
- Icon representation

## Usage

### Basic Usage

```dart
import 'package:ngerekrut/screens/full_chat_screen.dart';

// In your app
MaterialApp(
  home: FullChatScreen(
    currentUserId: 'user_123',
    currentUserName: 'Demo User',
  ),
)
```

### Sending Messages

```dart
// Text message (via composer)
await _chatController.insertMessage(Message.text(
  id: 'unique_id',
  authorId: 'user_123',
  text: 'Hello, World!',
  createdAt: DateTime.now(),
));

// System message
await _chatController.insertMessage(Message.system(
  id: 'unique_id',
  authorId: 'system',
  text: 'User joined the chat',
  createdAt: DateTime.now(),
));

// Streaming message (AI response)
final streamMessage = Message.textStream(
  id: 'stream_123',
  authorId: 'assistant',
  streamId: 'stream_123',
  createdAt: DateTime.now(),
);
await _chatController.insertMessage(streamMessage);

// Update as stream progresses
await _chatController.updateMessage(
  streamMessage,
  Message.text(
    id: 'stream_123',
    authorId: 'assistant',
    text: 'Partial response...',
    createdAt: streamMessage.createdAt,
  ),
);
```

## Architecture

```
┌─────────────────────────────────────────┐
│         FullChatScreen (UI)             │
│  - Chat widget with custom builders     │
│  - Message options menu                 │
│  - User actions (pin, delete, reply)    │
└─────────────────────────────────────────┘
                    ↕
┌─────────────────────────────────────────┐
│      DuckDBChatController               │
│  - Implements ChatController            │
│  - In-memory cache + DuckDB persistence │
│  - Operations stream for UI updates     │
└─────────────────────────────────────────┘
                    ↕
┌─────────────────────────────────────────┐
│       MessageRepository                 │
│  - CRUD operations for messages         │
│  - Reaction management                  │
│  - Query methods (search, filter, etc.) │
└─────────────────────────────────────────┘
                    ↕
┌─────────────────────────────────────────┐
│    ChatDatabaseService (DuckDB)         │
│  - Database connection management       │
│  - SQL execution                        │
│  - Transaction support                  │
└─────────────────────────────────────────┘
```

## Database Schema

### Messages Table
```sql
CREATE TABLE messages (
  id VARCHAR PRIMARY KEY,
  type VARCHAR NOT NULL,
  author_id VARCHAR NOT NULL,
  reply_to_message_id VARCHAR,
  created_at BIGINT,
  deleted_at BIGINT,
  failed_at BIGINT,
  sent_at BIGINT,
  delivered_at BIGINT,
  seen_at BIGINT,
  updated_at BIGINT,
  pinned BOOLEAN DEFAULT FALSE,
  status VARCHAR,
  text_content TEXT,
  media_source VARCHAR,
  media_metadata JSON,
  custom_metadata JSON,
  FOREIGN KEY (author_id) REFERENCES users(id)
)
```

### Reactions Table
```sql
CREATE TABLE reactions (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY,
  message_id VARCHAR NOT NULL,
  reaction_key VARCHAR NOT NULL,
  user_id VARCHAR NOT NULL,
  FOREIGN KEY (message_id) REFERENCES messages(id),
  FOREIGN KEY (user_id) REFERENCES users(id),
  UNIQUE (message_id, reaction_key, user_id)
)
```

## Notes

1. **Database Path**: DuckDB database disimpan di app documents directory
2. **Message Persistence**: Semua messages otomatis tersimpan ke DuckDB
3. **Soft Delete**: Delete messages menggunakan soft delete (deleted_at timestamp)
4. **Stream State**: Untuk streaming messages, stream state harus dikelola secara eksternal
